# `config`コマンド概要

gitの設定を管理するコマンド。

とりあえずこれでユーザー情報を設定しないとコミットができないので、
gitを使ったことがある人なら一度は使ったことがあると思う。  
ただ、一度設定してしまえば変更することは少ないので、
普段意識してなくて環境構築時に毎回つっかかる項目の印象がある。

## スコープ

設定は以下の3段階のスコープを持つ。
- local: プロジェクトの設定
- global: ユーザー内で共有される設定
- system: システム全体で共有される設定

各設定の値を参照するときは`local > global > system > デフォルトの値`の順に参照していく。

## My設定項目

毎回必要な時に調べて設定してるのでメモとして残す。  
ゴチャゴチャと設定してしまうと自分の開発環境以外の時に困ってしまうので、
出来るだけ設定をしない方針にしてる。

```git:.gitconfig
[user]
        name = yosshi-4989
        email = archer0729@icloud.com
[core]
        # 編集があるときにvimを利用する
        editor = vim -c \"set fenc=utf-8\"
        # コメント行にする文字を`#`から変更する
        # コメントのprefixに`#チケット番号`をよく使うのでコメント編集等の時に使いやすくする
        commentChar = ";"
```

設定コマンドは以下です。  
※当たり前ですが、`user.name`と`user.email`は適宜変更してください。

```bash
git config --global user.name "yosshi-4989"
git config --global user.email "archer0729@icloud.com"
git config --global core.editor 'vim -c "set fenc=utf-8"'
git config --global core.commentChar ';'
```

# よく使うオプション

## --local, --global

設定するスコープを指定するオプション。`--system`は使ったことが無い。  
基本globalで設定することが多いけど、たまにリポジトリ単位でユーザーを変えたいことがある…かなぁ？

## -l, --list

設定値の一覧を確認する。  
スコープの指定が無い場合、全てのスコープの設定が出るので指定することを推奨する。  
分かるのであれば直接設定ファイルを開いても同じなのでお好みで。

# コメント

ただ、新環境で必ず使って毎回調べるので備忘録として残せたのはよかった。
