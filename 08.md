# `stash`コマンド概要

作業中の差分を一時的に退避するコマンド。  
ステージングしていない新規ファイルは`stash`の対象外になるため、新規ファイルを対象にしたい場合はステージングしておくことで一緒に退避することができる。

さらにコマンドを指定することで退避した差分コマンドに関しての操作ができる。  
`stash`したファイル差分は`stash@{<x>}`という名前でスタックしていくので、
必要に応じてコマンドの対象に指定する。
以下は自分がよく使うコマンド。

|コマンド|操作|
|-------|----|
|list|退避したファイル差分の一覧表示|
|show|退避したファイル差分の詳細表示|
|drop|退避したファイル差分を削除|
|pop/apply|退避したファイル差分をHEADに適用する|
|(push)|差分ファイルを退避する|

# よく使うオプション

## `git stash`時のオプション

### -p, --patch

退避の対象を差分単位でステージングするかどうかをインタラクティブに選択できるオプション。  
仕様は`add`(02\.md)の`-p`オプションとほぼ同じなので入力方法は割愛。

## `git stash show`時のオプション

オプション指定がない場合はファイル名とその差分行数のみが表示される。  
`git stash show`のオプションは`git diff`のオプションを使いまわせる。

### -p, -u, --patch

ファイルごとの差分の詳細表示。  
具体的にどんな差分があるかを確認できる。
