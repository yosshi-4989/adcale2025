# 注意

`Git v2.23`から`checkout`が持っていた機能と同等の機能を持つ`switch`と`restore`の2つのコマンドが出来ました。  
ここでは`checkout`コマンドについて記載しますが、上記のコマンドを利用することを推奨します。

なお、ここでは`restore`と同様の使い方に関しての説明を行います。  
※`restore`コマンドを使ったことが無いので予想ではあります。

# `checkout`コマンド概要

`HEAD`の位置の変更とファイルの操作を行うコマンド。  
`--`オプションでファイル/ディレクトリを指定するとファイル操作になる。
ここではファイルパス/ディレクトリパスを指定するファイル操作について説明する。

`git checkout [<options>] [<commit/branch/tag>] [--] </path/to/dir_or_file>`でGitで管理されている指定したパスのファイル(ディレクトリの場合はそのディレクトリ配下のファイル全て)を指定のコミット時点のファイルに修正する。  
コミットが指定されない場合は`HEAD`+ステージングしたファイルをベースにするため、ステージングされていない差分を元に戻す。

地味に使う機会がそこそこある機能。  
開発が終わった後の残った差分を綺麗にしたり、仮実装して不要になった差分を消したり、
別ブランチで進めてた仮実装を本ブランチに反映させたりとたまに欲しくなる。
